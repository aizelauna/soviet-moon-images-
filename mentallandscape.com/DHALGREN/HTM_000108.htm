Date: 1994 Jan 25, 11:16:40 a.m. PST
From: Dredful (#96)
To:   *Core-Updates (#108)
Subj: Integrating exit update

I've just added exit integration into the core $exit and $room.

So, for every room and exit, you can set it up just about any way you like.

On $exit, you can set .look_msg to be the message appended to the rooms description. There are substitutions you can make as well. However, the exits .look_msg will only be appended if exit.obvious=2.

On $room, you can set .look_sep_msg to be the space between integrated exit messages. Usually the default is fine. But you could do whatever you want. You can also set .exit_display. When this property is true, not only will exits with .obvious=2 be put in the description, but those with obvious=1 will be displayed in this format:

Obvious exits: north, south, etc.

Tagged onto the end of the description. Maybe not very VR, but useful at times.

Of course, all this information can be found by:

help $room AND help $exit

Okay, so like I updated the Edit Options to include the `no parens' option.

So when you have it off, when you enter the verb editor, you'll see every parentheses that is in the verb code. When you have the option on, you'll see only those parentheses which are actually necessary to the running of the verb.
